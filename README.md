# Джарвис — голосовой помощник для Windows

Минимальный голосовой ассистент на C# (.NET 10). Слушает микрофон, распознаёт речь через бесплатный endpoint Google Speech API, выполняет команды и отвечает голосом (Windows TTS).

## Требования

- Windows 10/11
- .NET 10 SDK
- Микрофон
- Установленный русский голос Windows (для голосовых ответов)
  - Параметры → Время и язык → Речь → Управление голосами → Добавить → Русский

## Запуск

```powershell
cd C:\Users\berhard\Desktop\jarvis
dotnet run
```

После запуска Джарвис скажет «Джарвис в сети. Готов к работе.» — говорите команды. `Ctrl+C` для выхода.

## Поддерживаемые команды

| Сказать | Что сделает |
|---|---|
| «привет» | Поприветствует |
| «который час» / «время» | Скажет текущее время |
| «какое сегодня число» | Скажет дату |
| «открой браузер» | Откроет Google в браузере |
| «открой ютуб» | Откроет YouTube |
| «найди котики» | Поиск Google по запросу |
| «громче» / «тише» | Изменит громкость на 10% |
| «громкость» | Скажет текущий уровень |
| «выключи звук» / «включи звук» | Mute/unmute |
| «заблокируй» | Заблокирует ПК |
| «выключи компьютер» | Выключение через 60 сек |
| «перезагрузка» | Перезагрузка через 60 сек |
| «отмена выключения» | Отменить выключение |
| «выход» / «пока» | Завершить работу |

## Архитектура

```
Program.cs              — главный цикл
MicrophoneRecorder.cs   — захват микрофона + VAD (NAudio)
ISpeechRecognizer.cs    — интерфейс распознавания
GoogleSpeechRecognizer  — реализация: бесплатный Google endpoint
ISpeechSynthesizer.cs   — интерфейс TTS
WindowsSpeechSynthesizer — реализация: System.Speech
CommandHandler.cs       — диспетчер команд
```

## Замена сервиса распознавания (Google → Azure)

В `Program.cs` поменять одну строку:

```csharp
ISpeechRecognizer recognizer = new GoogleSpeechRecognizer();
// →
ISpeechRecognizer recognizer = new AzureSpeechRecognizer(apiKey, region);
```

Класс `AzureSpeechRecognizer` реализует тот же интерфейс `ISpeechRecognizer`. Остальной код не меняется.

## Настройка чувствительности микрофона

В `MicrophoneRecorder.cs`:

- `RmsThreshold` (0.020 по умолчанию) — порог громкости для старта записи. Если Джарвис не реагирует — уменьши, если ловит шум — увеличь.
- `SilenceMsBeforeStop` (1000 мс) — сколько тишины ждать после фразы перед отправкой на распознавание.

## Известные ограничения

- Используется неофициальный endpoint Google Speech (тот же, что в Python `SpeechRecognition`). Может ломаться без предупреждения. План — добавить `AzureSpeechRecognizer`.
- Continuous-listening без wake-word: Джарвис реагирует на любую речь после паузы. Для wake-word «джарвис» нужно добавить фильтр в `CommandHandler.Handle`.
